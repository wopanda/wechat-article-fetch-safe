# wechat-article-fetch-safe 开发记录

## 2026-03-26 改造总览

### 本轮目标
把当前仓库从“能抓微信公众号文章的 beta 版”推进成一条更完整、更可解释的抓取链：

1. 用户给一个公众号链接
2. 系统按固定顺序决定怎么抓
3. 遇到不同页面情况时有清晰分流
4. 最后保留 OCR 作为条件性兜底
5. 输出结构化结果，能解释这次到底走了哪一层

核心目标不是重写，而是把这件事做成：
**HTTP → Browser → OCR（最后兜底）** 的明确决策链。

## 第一阶段：工程化基础补强

### 已完成

#### 1. 主脚本结构化失败分类
在 `scripts/fetch_wechat_article.py` 中补了：
- `ExtractFailure`
- 页面信号识别：
  - `verify_required`
  - `captcha_or_env_check`
  - `article_deleted`
  - `content_not_found`
  - `anti_bot_suspected`
- 成功 / 失败统一 JSON 输出

#### 2. 浏览器 fallback 从固定动作升级为带状态观察
浏览器模式新增：
- 页面高度观察
- 文本长度观察
- 图片数量观察
- 稳定轮次收敛

#### 3. 调试产物保留
`--debug --artifacts-dir` 模式下保存：
- `page.html`
- `meta.json`
- `browser_final.png`

用途：
- 判断 verify/captcha
- 判断 DOM 结构变化
- 区分“页面访问问题”与“提取逻辑问题”

#### 4. 最小回归框架
新增：
- `scripts/run_regression.py`
- `tests/fixtures/urls.json`
- `tests/results/`

这让后续修改可以基于同一批 URL 回归，而不是只靠手工感受。

#### 5. 修正浏览器 fallback 误触发
在实测中发现 `should_browser_fallback()` 误把 `page_status` 当成 `quality_metrics` 子字段读取，导致 HTTP 已足够好时仍误切浏览器。

已修复为读取 candidate 顶层 `page_status`。

## 第三阶段：抓取决策链定型

### 已完成

#### 1. 决策链正式明确
当前主链已经定型为：

1. **HTTP**：`wechat-mobile` / `chrome-desktop`
2. **Browser fallback**：`playwright-mobile`
3. **OCR 最后兜底**：`ocr-fallback`

并新增输出字段：
- `used_browser_fallback`
- `used_ocr_fallback`
- `decision_path`

这意味着现在返回结果时，不只告诉“抓到了没”，还会告诉：
- 这次最终是哪个策略成功
- 中间都尝试了哪些策略

#### 2. OCR 兜底接入
新增 OCR 相关能力：
- `crop_center_image()`
- `run_tesseract_ocr()`
- `normalize_ocr_text()`
- `should_try_ocr()`
- `build_ocr_candidate()`

OCR 当前定位很明确：
- 不是默认第二步
- 不是用来绕过 verify / anti-bot
- 只在 **Browser 页面视觉可见但 DOM 提取明显不足** 时尝试

#### 3. OCR 首个真实 bug 修复
在实测 OCR 时发现：
- tesseract 输出 txt 路径计算错误

已修复。

随后 OCR 能真实执行，但若 OCR 结果本身过短，则会返回：
- `ocr_failed`
- 原因：`OCR 结果过短，未达到可用阈值`

这说明 OCR 已真正接入，而不是纸面设计。

## 关键实测记录

### 样例 1：腾讯“龙虾”文章
URL：
`https://mp.weixin.qq.com/s/7scfaabhJETu42G-AkpZ8Q`

结果：
- `success_http`
- `fetch_method = wechat-mobile`
- `used_browser_fallback = false`
- `used_ocr_fallback = false`

说明：
- 正常文章仍优先走 HTTP 主链
- 新增 Browser / OCR 后没有把主链搞坏

### 样例 2：深圳 OPC 社区文章
URL：
`https://mp.weixin.qq.com/s/1GWJoOLp4zpOs7SZk2ra1Q`

实际决策路径：
- `wechat-mobile`
- `chrome-desktop`
- `playwright-mobile`
- `ocr-fallback`

最终结果：
- `success_browser`
- Browser 拿到标题 / 作者 / 发布时间
- OCR 已真实触发，但 OCR 结果过短，没有覆盖 Browser 结果

说明：
- 决策链已经完整跑通
- OCR 已经能被条件触发
- OCR 不是万能补丁，只会在结果足够好时升级成最终结果
